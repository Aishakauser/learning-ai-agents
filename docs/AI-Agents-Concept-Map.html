Definitions

Toolkit

VS Code

A free code editor (like Word, but for code) that colors your code, autocompletes, and flags errors. Add-ons called extensions teach it new tricks. It has a built-in terminal too.

↑ back to map
Toolkit

Terminal (command line)

A text-based way to control your computer by typing commands instead of clicking. Faster and more precise for developer tasks; most programming tools expect it.

↑ back to map
Toolkit

Git

A tool that takes snapshots (called commits) of your project over time, so you can see what changed and rewind if you break something. Like unlimited save slots in a video game.

↑ back to map
Toolkit

GitHub

A website that stores a copy of your git project in the cloud so you can share it and collaborate. Git is the camera; GitHub is the shared photo album.

↑ back to map
Toolkit

GitHub CLI (gh)

GitHub's command-line tool. gh auth login signs you in through the browser; then one command can create a repo and upload your code — no manual tokens.

↑ back to map
Toolkit

Virtual environment

An isolated bubble of installed packages for one project, so different projects' dependencies don't collide. Each job carries its own toolbox.

↑ back to map
Toolkit

API key

A secret password that proves you're allowed to use a paid service (like Claude's models). Kept in a hidden .env file and never shared publicly.

↑ back to map
Tech stack

Front-end

Everything the user sees and touches — buttons, screens, forms. The dining room of the restaurant.

↑ back to map
Tech stack

Back-end

The hidden logic that processes requests and enforces rules. The kitchen where orders are cooked. Agents live here.

↑ back to map
Tech stack

Database

Where information is stored permanently — users, messages, orders. The pantry and fridge.

↑ back to map
Tech stack

Vector database

A special database that stores the meaning of text as lists of numbers, so you can search by similarity of meaning rather than exact words. The backbone of RAG.

↑ back to map
Tech stack

Python

A readable programming language that dominates AI because nearly every AI tool is built for it first. The default language for agents.

↑ back to map
The LLM

Neural network

A pattern-learning program loosely inspired by brain cells. You don't program rules; you show it millions of examples and it learns the patterns by adjusting internal numbers (weights).

↑ back to map
The LLM

LLM (Large Language Model)

A huge neural network trained on text that predicts the next chunk of text, over and over. Like phone autocomplete scaled up a billionfold. The "brain" of an agent.

↑ back to map
The LLM

Token

A chunk of text the model reads — often a word or part of a word (≈ ¾ of a word). You pay per token, in and out.

↑ back to map
The LLM

Context window

The maximum amount of text (in tokens) the model can consider at once — its "desk." Instructions, history, documents, and your question all must fit on it.

↑ back to map
The LLM

Embedding

A list of numbers representing the meaning of a piece of text. Similar meanings get similar numbers — which is how "search by meaning" (and RAG) works.

↑ back to map
The LLM

System prompt

A special instruction that sets the model's role and rules for the whole conversation ("You are a careful research assistant…"). Your most powerful lever.

↑ back to map
The LLM

Hallucination

When the model states something false with confidence, because it predicts plausible text rather than checking facts. Fixed by giving it real sources (RAG) and tools.

↑ back to map
Core

Agent

An LLM given a goal that decides for itself what steps to take and which tools to use, looping until done. Formula: LLM + loop + tools + memory + guardrails.

↑ back to map
Core

Agent loop

The beating heart of an agent: THINK (what next?) → ACT (call a tool) → OBSERVE (read the result) → repeat until the goal is met.

↑ back to map
Core

Tools / function calling

Functions the model is allowed to call to act on the world — search the web, do math, query a database. The model requests a tool; your code runs it and returns the result. This is what turns an LLM into an agent.

↑ back to map
Core

Memory

Short-term (the current conversation, re-sent each turn) and long-term (facts saved outside the model and pulled back when relevant). LLMs are forgetful by default, so memory is bolted on.

↑ back to map
Core

Guardrails

Rules and limits that keep an agent safe and on-task — input/output checks, blocking risky actions without approval, and caps on how many loops it can run. A human-in-the-loop checkpoint is the most important one.

↑ back to map
Capable

RAG (Retrieval-Augmented Generation)

Letting an agent answer from your documents: retrieve the most relevant chunks (via embeddings), add them to the prompt, then the model writes its answer from them. Fixes hallucination and stale knowledge.

↑ back to map
Capable

Planning

Breaking a big goal into smaller steps before acting (task decomposition). Often a to-do list the agent maintains and updates as it works.

↑ back to map
Intelligent

ReAct (Reason + Act)

The agent writes out its reasoning before each action, then acts, then reasons about the result. Thinking out loud improves decisions and makes behavior debuggable.

↑ back to map
Intelligent

Reflection / self-correction

After producing a result, the agent reviews its own work against the goal, finds flaws, and tries again. The "draft, then edit" pass.

↑ back to map
Intelligent

Evals (evaluation)

A test set of tasks with known good answers, used to measure whether the agent actually performs well — like unit tests for behavior. Without them you're flying blind.

↑ back to map
Intelligent

Context engineering

Curating exactly what goes into the context window at each step — sharp instructions, only relevant documents, summarized history. Widely considered the core skill of building good agents.

↑ back to map
Scaling

Framework

Pre-built structure that handles the repetitive plumbing (the tool loop, memory, model connections) so you write only what's unique to your project. Optional — you can start with none.

↑ back to map
Scaling

LangChain

The most popular agent framework — a big toolbox of pre-made components and integrations. Great for prototyping; can feel heavy because it does so much.

↑ back to map
Scaling

LangGraph

Build agents as controllable flowcharts (graphs): you define the allowed paths, the agent decides within them. Preferred for reliable, production agents.

↑ back to map
Scaling

Claude Agent SDK

Anthropic's official toolkit for building agents on Claude — packages the loop, tools, memory, sub-agents, and permissions. Battle-tested from Anthropic's own coding agent.

↑ back to map
Scaling

MCP (Model Context Protocol)

An open standard for plugging tools and data sources into an AI in a uniform way — "USB-C for AI tools." One standard plug, many devices.

↑ back to map
Scaling

Multi-agent system

Several specialist agents plus a coordinator (orchestrator) that delegates and combines results — like a newspaper editor assigning stories to reporters. Each sub-agent gets its own clean context.

↑ back to map
Scaling

Deep Agent

An agent built for long, complex tasks. Four ingredients: explicit planning, a file system for memory, sub-agents with clean contexts, and detailed instructions/skills. Just Parts 4–7 assembled thoughtfully.

↑ back to map
Scaling

Hermes Agent

Nous Research's open-source, self-hosted Deep Agent: persistent memory, self-written skills, built-in tools, runs on chat platforms. A real-world example of the Deep Agent pattern. (Note: "Hermes" is also a separate family of LLMs — Hermes Agent is the agent, not the model.)
